Sandbox isolation (these should be blocked)

Known not to work (by design)

The iframe runs without allow-same-origin so the page has an opaque origin. This blocks:

  • localStorage, sessionStorage, IndexedDB — storage APIs throw SecurityError
  • document.cookie — throws SecurityError
  • window.parent.* reads — cross-origin blocked (postMessage still works)
  • Vault-relative URLs like <img src="attachments/foo.png"> — no base URL points to the vault; use absolute HTTPS or data URLs
  • Service workers, geolocation, clipboard, notifications, and most permission-gated APIs
